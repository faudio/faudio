12  Little Matrix Machine (LMM)
13 
14  - Matrix registers: [0..kLmmRegs]
15 
16  - Each matrix register contains a raw buffer of known size.
17 
18  - Buffers are allocated using the alloc opcode.
19 
20  - Initially, all buffers have *max size* and *size* 0.
21  - Buffer *max size* is constant until swap, or alloc is called.
22  - Buffer *size* is affected by data ops as follows:
23  - Using copy, split, ap1 or ap2 with a too small buffer will
24  fail.
25  - All other opcodes use vector arithmetic, truncating to the
26  shortest argument size.
27 
28  - Convenience registers:
29  off, step, loop
30 
31  - Instructions
32  - Unfolding operations stores result in second operand while folding
33  operations (most) store in the first.
34 
35  - Size arguments are in bytes.
36 
37 
38  - Full table
39  alloc s r
40  swap r1 r2
41

43  split s r1 r2 (r1[0..s-1],r1[s..n-1]) ==> (r1,r2)
44 
45  set T1 x r1 r1[i] <== x where i <- [0..n]
46  ap1 T1 f ct r1 r1[i] <== f(ct, r1[i]) where i <- [0..n]
47  ap2 T1 T2 f ct r1 r2 r1[i] <== f(ct, r1[i], r2[i]) where i <- [0..n]
48 
49  inc T1 r1 r1 <== r1 + 1
50  dec T1 r1 r1 <== r1 - 1
51  neg T1 r1 r1 <== r1 * (-1)
